# nsz browser toolkit

Веб-инструмент для работы с файлами форматов `.nsz` и `.nsp` (компрессия и декомпрессия) непосредственно в браузере.  
Всё выполняется локально, без отправки данных на сервер – используется Pyodide для запуска Python-библиотеки `nsz`.

## Возможности

- **Analyze** – отобразить подробную информацию о файле (структура, заголовки).
- **Verify** – проверить целостность файла.
- **Decompress** – распаковать `.nsz` в `.nsp`.

Поддерживаются два режима ввода:

1. **Режим файлов** – вы выбираете один или несколько файлов (через стандартный диалог). Они копируются в оперативную память, после чего с ними можно работать.  
   *Ограничение*: файлы размером более 2 ГБ не могут быть загружены в память и будут пропущены.

2. **Режим папки** – вы выбираете папку, которая монтируется через File System Access API. Файлы не копируются в память, а читаются напрямую.  
   *Примечание*: файлы размером более 2 ГБ могут вызывать ошибки доступа при обходе папки.

Результат распаковки:
- в режиме файлов – скачивается как обычный файл;
- в режиме папки – сохраняется в корень выбранной входной папки (если не выбрана отдельная выходная папка) и синхронизируется с реальной файловой системой.

## Ограничения

- **Сжатие (Compress) не поддерживается** – из-за использования многопоточности и низкоуровневых функций, недоступных в браузерном окружении Pyodide.
- **Файлы >2 ГБ** в режиме файлов игнорируются; в режиме папки могут вызывать ошибки доступа.
- Для работы требуется современный браузер с поддержкой:
  - WebAssembly;
  - File System Access API (для монтирования папок) – Chromium‑основанные браузеры (Chrome, Edge).

## Как пользоваться

1. Нажмите одну из кнопок действия: **Analyze**, **Verify** или **Decompress**.  
   При первом нажатии начнётся загрузка Pyodide и библиотеки `nsz` (занимает некоторое время, прогресс отображается в статусной строке).
2. Выберите файлы (режим файлов) или папку (режим папки). В режиме файлов можно выбрать несколько файлов.
3. При необходимости включите опцию `--fix-padding` – она применима к командам Verify и Decompress.
4. Дождитесь завершения операции. Результат появится в области вывода, а файлы – либо будут скачаны, либо сохранены в выбранной/входной папке.

## Решённые проблемы

В процессе разработки были устранены следующие трудности:

- **Автоматическая загрузка Pyodide** – не требуется отдельная кнопка «Load», всё происходит при первом действии.
- **Сохранение выбора файлов** – при переключении между режимами выделение не сбрасывается (используются отдельные множества для имён файлов и папок).
- **Ошибка `syncfs` для вложенных папок**  - отказ от создания подпапки `input/output`; теперь результат пишется в корень входной папки, что соответствует простейшему случаю из документации и гарантирует появление файла на диске.
- **Несовместимость опций командной строки с Python API**  - убраны опции, которые не принимаются функциями `nsz.verify` и `nsz.decompress` (например, `--quick`, `--keep`, `--overwrite`). Оставлен только рабочий флаг `--fix-padding`.
- **Работа с папками, выбранными до загрузки nsz** – они корректно монтируются после завершения загрузки, а статус «ожидает монтирования» в селекторе обновляется.
- **Обработка файлов >2 ГБ** – в режиме файлов они отсеиваются с предупреждением, в режиме папки попытка получить такой файл приводит к выводу ошибки.
- **Удаление кнопки Compress** – убрана, так как сжатие в браузере неработоспособно.

## Зависимости
